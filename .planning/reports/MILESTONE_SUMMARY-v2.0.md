## 1. Overview
Milestone v2.0 transformed **supportLegal** from a backend-heavy RAG prototype into a user-facing legal assistant. The primary focus was on implementing a high-performance, stateless frontend-backend interaction loop that supports streaming responses, conversational context, and authoritative legal reranking.

## 2. Architecture & Design
The system now follows a **Stateless Context Loop** architecture:
- **Frontend**: A Next.js 14 dashboard using a split-screen layout. It maintains a sliding window of the last 4 chat messages in local state.
- **API Protocol**: Switched from standard JSON (`/ask`) to Server-Sent Events (`/stream`).
- **Context Handling**: The backend uses a specialized "Query Rewriter" step (Groq/Llama-3) to contextualize follow-up questions (e.g., "Còn đối với trẻ em?") into full search queries.
- **RAG Core**: Article-level hybrid retrieval (Vector + BM25) with a legal authority priority layer.

### Data Flow
1. **User Query** → Frontend (appends history) → **Backend (`/stream`)**.
2. **Backend** → Rewrite Query → **Parallel Search (Qdrant + SQLite)**.
3. **Frontend** ← Yields **Citations** immediately (first frame).
4. **Backend** → **LLM Generation (Groq/Llama-3.3)** → **Parallel Classification**.
5. **Frontend** ← Yields **Tokens** + **Classification Badges** dynamically.

## 4. Key Decisions & Rationale
| Decision | Rationale |
|----------|-----------|
| **Groq as Primary** | Gemini and DeepSeek exhibited higher latency and more frequent rate limiting during peak loads. Groq 70B provided the best speed/quality trade-off for IRAC generation. |
| **Stateless Frontend** | Avoided backend session management (Redis/Postgres) to keep infrastructure simple and horizontally scalable. |
| **Article-Parent Retrieval** | Legal clauses (Khoản) lack context without their parent Article (Điều). Returning the full article ensures the LLM sees the complete legal frame. |
| **Legal Authority Priority** | In Vietnamese law, higher-level documents (Luật) override lower-level ones (Nghị định). Reranking ensures the most authoritative source is listed first. |

## 6. Known Tech Debt & Risks
- **LLM API Quotas**: High concurrency (>10 users) causes 429 errors from Groq. **Recommendation**: Implement semantic caching and local classifier (Ollama/Qwen).
- **Indexing Scale**: Full 518K document re-indexing is slow on CPU. **Recommendation**: Move to GPU-enabled environment for Phase 10 completion.
- **Reranker Latency**: The BAAI Cross-Encoder is currently disabled in some paths due to Windows DLL issues.

## 7. Getting Started (Team Onboarding)
To run the full v2.0 stack:
1. **Backend**: 
   - `pip install -r requirements.txt`
   - `python main.py` (ensure `.env` has `GROQ_API_KEY`)
2. **Frontend**:
   - `cd frontend && npm install`
   - `npm run dev`
3. **Verify**: Open `http://localhost:3000` and ask: "Hành vi trộm cắp bị xử lý thế nào?".
