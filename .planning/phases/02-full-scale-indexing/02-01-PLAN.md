# Plan: 02-01 - Background Indexer Implementation

This plan covers the implementation of a robust, streaming background indexer for the 3.6GB Vietnamese legal dataset.

## Proposed Changes

### Core Logic

- **Streaming Ingestion**: Use Hugging Face `datasets` with `streaming=True` to avoid OOM.
- **Hybrid Chunking**:
    - Primary: `MarkdownHeaderTextSplitter` to preserve legal structure (Chapters, Sections, Articles).
    - Secondary: `RecursiveCharacterTextSplitter` for chunks > 1000 characters.
- **Batched Processing**: Embed and upsert in batches of 64/100 to optimize throughput.

## Success Criteria

1.  **Streaming**: Script handles continuous data flow without significant RAM growth.
2.  **Structure**: Vectors in Qdrant retain metadata about Law headers.
3.  **Persistence**: Data survives script termination and restarts.

## Verification Plan

### Automated Tests
- `pytest tests/test_indexer.py`: Verify splitter logic and batching boundaries.

### Manual Verification
- Run `python indexer.py --limit 10` and inspect Qdrant Dashboard/Payload.
