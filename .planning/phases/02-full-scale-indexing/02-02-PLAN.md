# Plan: 02-02 - Progress Tracking and Idempotency logic

This plan ensures that the indexing process is resumable and reliable over long periods.

## Proposed Changes

### State Management

- **SQLite Tracking**: Persistent `indexing_status` table to store `doc_id` of successfully processed documents.
- **Commit Strategy**: Mark as processed only *after* successful Qdrant/SQLite insertion to ensure data integrity.
- **Logging**: Use `tqdm` for human-readable progress and `logging` for audit trails.

## Success Criteria

1.  **Resumability**: Interrupting the script and restarting results in skipping already processed documents.
2.  **Observability**: Clear logs showing progress and any document failures.

## Verification Plan

### Automated Tests
- Test idempotency by mocking the DB and checking skip logic.

### Manual Verification
- Start indexer, kill it (Ctrl+C), restart, and verify "Skipped" counts match previous progress.
