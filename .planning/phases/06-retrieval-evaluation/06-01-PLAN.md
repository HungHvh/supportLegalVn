# Phase 6 Plan: Retrieval Evaluation

## 📝 Overview
This plan implements a robust evaluation pipeline to compare the Phase 5 "Optimized" RAG system against a "Baseline" version. We will generate a golden dataset of legal queries and use Ragas with gemini-2.0-flash to score retrieval and generation quality.

## 🏗 Tasks

### 1. Environment & Dependencies
- [ ] Install required libraries: `ragas`, `langchain-google-genai`, `pandas`, `tabulate`.
- [ ] Verify `GOOGLE_API_KEY` is correctly set in `.env` for the evaluation scripts.

### 2. Synthetic Dataset Generation
- [ ] Create `scripts/generate_eval_data.py`.
    - [ ] Initialize `RagDatasetGenerator` using `gemini-2.0-flash`.
    - [ ] Target the "Luật Doanh nghiệp" domain in `legal_poc.db`.
    - [ ] Generate 30-40 (Question, Context, Ground Truth) triplets.
    - [ ] Save to `.planning/phases/06-retrieval-evaluation/golden_set_synthetic.json`.

### 3. Manual Edge Case Curation
- [ ] Create `.planning/phases/06-retrieval-evaluation/golden_set_manual.json`.
    - [ ] Define 10-15 edge case questions (Temporal, Intersection, Exceptions).
    - [ ] Provide expert-level ground truth answers and context IDs.

### 4. Ablation Runner Implementation
- [ ] Create `scripts/run_benchmarks.py`.
    - [ ] Implement `AblationPipeline` that wraps `LegalRAGPipeline`.
    - [ ] Mode A (Baseline): No hybrid, no classifier, no breadcrumbs.
    - [ ] Mode B (Optimized): Full Phase 5 features.
    - [ ] Loop through the Golden Set and collect (Query, Baseline_Response, Optimized_Response, Contexts).

### 5. Ragas Scoring & Reporting
- [ ] Implement scoring logic in `scripts/run_benchmarks.py`.
    - [ ] Configure `LangchainLLMWrapper` for Gemini.
    - [ ] Compute: Faithfulness, Answer Relevance, Context Precision, Hit Rate @ 3.
- [ ] Generate `eval_report.md` (Markdown summary) and `eval_results.csv` (Detailed logs).

## ✅ Verification Loop

### UAT-06-01: Dataset Quality
- [ ] Run `generate_eval_data.py` and manually verify 5 samples for legal accuracy.

### UAT-06-02: Pipeline Comparison
- [ ] Run `run_benchmarks.py` on a subset (5 queries).
- [ ] Confirm that "Baseline" and "Optimized" are returning different nodes/responses as expected.

### UAT-06-03: Final Report
- [ ] Execute full benchmark (50 queries).
- [ ] Verify `eval_report.md` contains the comparison table and metric deltas.

## 📅 Timeline Estimate
- **Development**: 4-6 hours.
- **Execution**: 1-2 hours (including API wait times and manual verification).
