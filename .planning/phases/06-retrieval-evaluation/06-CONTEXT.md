# Phase 6 Context: Retrieval Evaluation (v1.0)

## 🎯 Objective
Establish a quantitative and qualitative benchmark for the hierarchical hybrid retrieval system compared to a standard dense retrieval baseline.

## ⚖️ Evaluation Strategy: Ablation Study
We will evaluate two configurations on the same dataset to measure the delta provided by Phase 5 optimizations:
1.  **Baseline**: Dense Vector Search only, no hierarchical context, no FTS5 fusion.
2.  **Optimized**: Hybrid Search (Dense + Sparse), Hierarchical Context Injection, and Domain Classification.

## 📊 Metrics
| Metric | Goal | Description |
| :--- | :--- | :--- |
| **Hit Rate @ 3** | > 85% | Does the ground truth context appear in top 3 results? |
| **MRR** | > 0.7 | Mean Reciprocal Rank (Higher is better). |
| **Faithfulness** | > 0.9 | (Ragas) Ensure the answer is derived strictly from context. |
| **Answer Relevance** | > 0.8 | (Ragas) Ensure the answer directly addresses the query. |

## 🧪 Dataset Composition (The Golden Set)
- **Synthetic Breadth (30-40 Qs)**: Generated by Gemini-1.5-Flash scanning the "Luật Doanh nghiệp" corpus.
- **Manual Depth (10-15 Qs)**: Focus on:
    - **Temporal Conflicts**: Law 2014 vs Law 2020 (superseded clauses).
    - **Intersections**: Enterprise Law vs Criminal/Tax Law intersections.
    - **Exception Clauses**: Handling "Trừ trường hợp..." (Unless otherwise specified...) logic.

## 🛠 Technical Implementation
- **Judge**: Ragas configured with `Gemini-1.5-Flash` (via custom Langchain/VertexAI wrapper).
- **Scope**: Primary focus on **Enterprise Law (Luật Doanh nghiệp)** for POC validation.
- **Output**: 
    - `eval_report.md`: Visual summary with comparison tables and delta analysis.
    - `eval_details.csv`: Full trace of every query, retrieved contexts, and scores.

## 🛑 Blockers & Risks
- **Token Usage**: Generating 50+ evaluation triplets + Ragas scoring involves multiple LLM calls.
- **Ground Truth**: Manual verification of synthetic ground truth is recommended to ensure benchmark integrity.
