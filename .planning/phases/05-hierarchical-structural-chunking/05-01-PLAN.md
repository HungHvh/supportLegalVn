# Plan 05-01: Legal Structure Parser

Implement a robust, hierarchical parser for Vietnamese legal documents using Regex and Layout-aware Markdown.

## Context
Downstream RAG needs units of text that follow legal hierarchy (Article/Clause) rather than fixed token lengths.

## Tasks

### 1. Implement LegalNode Model
Create Pydantic models to represent the legal hierarchy.
<action>
Create `core/parser.py` with:
- `LegalLevel` Enum (PART, CHAPTER, SECTION, ARTICLE, CLAUSE, POINT).
- `LegalNode` Pydantic model containing:
    - `level`: LegalLevel
    - `title`: str (e.g., "Điều 1")
    - `content`: str
    - `children`: List[LegalNode]
    - `metadata`: Dict
</action>

### 2. Implement Regex Logic
Implement the core splitting logic using the researched patterns.
<action>
In `core/parser.py`, implement `VietnameseLegalParser` class:
- `split_by_pattern(text, level)`: Uses `re.finditer` with `MULTILINE | IGNORECASE`.
- Patterns:
    - PART: `r'^Phần\s+.*?\.'`
    - CHAPTER: `r'^Chương\s+.*?\.'`
    - ARTICLE: `r'^Điều\s+\d+.*?\.'`
    - CLAUSE: `r'^\d+\.'`
    - POINT: `r'^[a-z]\)'`
</action>
