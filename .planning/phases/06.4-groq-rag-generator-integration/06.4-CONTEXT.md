# Context: Phase 06.4 - Groq RAG Generator Integration

## Goal
Replace the Gemini-based RAG response generation with a Groq-based (Llama-3) generation flow to resolve budget constraints and quota limitations.

## Context
- Gemini API is currently used for generating the final IRAC response in `main.py` or wherever the RAG pipeline is defined.
- Gemini has been reporting quota exhaustion ("out of money").
- Groq is already integrated for classification (Phase 6.3) and has proven to be very fast and reliable.
- The system should now use Groq for both classification and answer generation.

## Technical Requirements
- Update the RAG pipeline to use Groq for the generation step.
- Ensure the prompt templates and citation logic remain intact.
- Handle Groq-specific parameters (model name, temperature, etc.).

## Verification Criteria
- [ ] RAG responses are generated by Groq (verified via logs/headers).
- [ ] Citations are correctly formatted and attributed.
- [ ] Latency is within acceptable limits (likely lower than Gemini).
