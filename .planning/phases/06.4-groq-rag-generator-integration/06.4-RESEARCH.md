# Research: Phase 06.4 - Groq RAG Generator Integration

## Technical Analysis

### Gemini vs Groq Interface
- **Gemini**: Uses `google-generativeai` SDK. Supports async and streaming via `generate_content_async(stream=True)`. Returns `AsyncGenerateContentResponse` chunks with `.text` property.
- **Groq**: Uses OpenAI-compatible REST API. `GroqClient` currently uses `httpx` but only for non-streaming calls.

### Streaming Implementation for Groq
To support streaming in `GroqClient`, we need to:
1. Use `httpx.AsyncClient().stream("POST", ...)` with `stream=True` in payload.
2. Iterate over lines.
3. Parse `data: {...}` lines.
4. Yield content.

Example pattern:
```python
async with client.stream("POST", url, json=payload, headers=headers) as response:
    async for line in response.aiter_lines():
        if line.startswith("data: "):
            data = json.loads(line[6:])
            content = data["choices"][0]["delta"].get("content")
            if content:
                yield content
```

### Pipeline Refactoring
The `LegalRAGPipeline` should be updated to a provider-agnostic initialization.

```python
def __init__(self, retriever, provider="gemini", model_name="gemini-2.0-flash"):
    self.retriever = retriever
    self.provider = provider
    self.model_name = model_name
    self.client = self._get_client(provider, model_name)
```
