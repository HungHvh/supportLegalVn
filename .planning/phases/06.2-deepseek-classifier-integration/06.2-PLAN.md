# Phase 6.2: DeepSeek API Classifier Integration - Plan

**Status:** Draft
**Phase:** 6.2
**Goal:** Integrate DeepSeek API to replace Qwen for question classification, with Gemini as a robust fallback.

## 🟢 Wave 1: Infrastructure & Clients

### Task 1.1: Create DeepSeek Client
- **File:** `tools/deepseek_client.py`
- **Action:**
  - Implement `DeepSeekClient` inheriting from `BaseLLMClient`.
  - Use `httpx` (async) to call `https://api.deepseek.com/chat/completions`.
  - Implement `generate_content_async` with the same signature as `GeminiClient`.
  - Wrap the response in a structure that provides a `.text` attribute for compatibility with `LegalQueryClassifier`.
  - Include basic retry logic (e.g., using `tenacity`) for transient API errors.
- **Read first:** `tools/llm_client_base.py`, `tools/gemini_client.py`
- **Acceptance criteria:**
  - `tools/deepseek_client.py` exists and contains `DeepSeekClient`.
  - `generate_content_async` successfully returns text from the DeepSeek API.

### Task 1.2: Add DeepSeek API Key to Environment
- **File:** `.env.example`, `.env`
- **Action:** Add `DEEPSEEK_API_KEY` placeholder.
- **Acceptance criteria:** `.env.example` contains `DEEPSEEK_API_KEY=`.

## 🟡 Wave 2: Core Integration

### Task 2.1: Update LegalQueryClassifier
- **File:** `core/classifier.py`
- **Action:**
  - Import `DeepSeekClient`.
  - Update `_get_client` to handle `provider_name == "deepseek"`.
  - Update `__init__` default `provider` to `"deepseek"`.
  - Update `__init__` default `fallback_provider` to `"gemini"`.
  - Ensure the failover loop correctly skips the failing provider and calls the next.
- **Read first:** `core/classifier.py`
- **Acceptance criteria:**
  - `LegalQueryClassifier` can initialize and use `deepseek` by default.
  - Failover to `gemini` works if `deepseek` fails.

### Task 2.2: Optimize Vietnamese Legal Prompt
- **File:** `core/classifier.py`
- **Action:** Refine the `system_prompt` strings to be more explicit for DeepSeek's Vietnamese understanding. Ensure the 6 domains are clearly described.
- **Acceptance criteria:** `system_prompt` updated with refined Vietnamese instructions.

## 🔴 Wave 3: Verification & Evaluation

### Task 3.1: Unit Tests for DeepSeek Integration
- **File:** `tests/test_deepseek_integration.py` [NEW]
- **Action:** Create mock tests to verify that `LegalQueryClassifier` calls the `DeepSeekClient` and handles the response correctly.
- **Acceptance criteria:** `pytest tests/test_deepseek_integration.py` passes.

### Task 3.2: Comparison Evaluation
- **Action:** Run `scripts/eval_pipeline.py` (if applicable for classification) or a manual test suite with 20-30 varied legal queries to compare DeepSeek classification against Gemini.
- **Acceptance criteria:** Classification accuracy is >= Gemini baseline on the test set.

## 🏁 Verification Strategy

### Automated Tests
- `pytest tests/test_deepseek_integration.py`
- `pytest tests/test_classifier.py` (existing tests)

### Manual Verification
- Trigger a classification call via the API or a scratch script.
- Intentionally block DeepSeek (e.g., wrong API key) to verify fallback to Gemini works.

## 🛡️ Threat Model
- **Threat:** API Key exposure.
- **Mitigation:** Use `.env` and `os.environ`. Never hardcode keys.
- **Threat:** Malicious query injection to bypass classification.
- **Mitigation:** The classifier output is strictly used for filtering; any hallucination is caught by the subsequent retrieval and generation stages.
