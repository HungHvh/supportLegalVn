## Technical Investigation

### DeepSeek API Specification
DeepSeek provides an OpenAI-compatible API, which simplifies integration with existing Python patterns.
- **Base URL:** `https://api.deepseek.com`
- **Endpoints:** `/chat/completions` (OpenAI compatible)
- **Model:** `deepseek-chat` (currently refers to DeepSeek-V3)
- **Authentication:** Standard Bearer Token in `Authorization` header.
- **Payload Structure:**
  ```json
  {
    "model": "deepseek-chat",
    "messages": [
      {"role": "system", "content": "..."},
      {"role": "user", "content": "..."}
    ],
    "stream": false,
    "response_format": { "type": "json_object" }
  }
  ```

### Integration Strategy
1.  **New Tool:** Create `tools/deepseek_client.py` inheriting from `BaseLLMClient`.
2.  **Implementation:** Use `httpx` or `requests` (async) to call the DeepSeek API. Since the project uses `GeminiClient` with a custom interface, we should mimic the `generate_content_async` method.
3.  **Classifier Update:**
    - Add `deepseek` to the provider mapping in `core/classifier.py`.
    - Update `__init__` to accept `deepseek` as primary.
    - Implement `json_object` response format handling (DeepSeek-V3 supports this).

### Vietnamese Legal Nuances
DeepSeek-V3 has been reported to have high proficiency in Vietnamese.
- **Prompting:** The system prompt should emphasize the specific 6 domains defined in Phase 3.
- **Output:** Enforce strict JSON output to match the `QueryClassification` Pydantic model.

## Existing Patterns & Reuse

### Pattern: LLM Client
Existing clients (`GeminiClient`, `QwenDashScopeClient`) wrap their respective SDKs but expose a common `generate_content_async` method. The `DeepSeekClient` should follow this for consistency with the `LegalQueryClassifier._call_provider` logic.

### Pattern: Failover
`LegalQueryClassifier.classify` already implements a retry loop over a list of providers. We will set the list to `["deepseek", "gemini"]`.

## Validation Architecture

### Component Level
- Unit test for `DeepSeekClient` using a mock server (e.g., `respx`).
- Integration test for `LegalQueryClassifier` with `deepseek` provider.

### End-to-End
- Run the `scripts/eval_pipeline.py` specifically for the classification task to compare DeepSeek performance against the baseline (Gemini).

## Risks & Mitigations
- **Risk:** API Latency/Overload. DeepSeek API sometimes experiences high demand.
- **Mitigation:** Ensure the 10s timeout and immediate failover to Gemini is robust.
- **Risk:** Response formatting. LLMs sometimes include markdown triple backticks.
- **Mitigation:** Use the existing `_parse_json_response` which handles markdown cleaning.
