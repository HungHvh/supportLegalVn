# Phase 6.1 Summary: Qwen Classifier Provider

## What Was Implemented
- Added a minimal provider contract in `tools/llm_client_base.py`.
- Added `QwenDashScopeClient` for DashScope-compatible Qwen calls.
- Added `QwenOllamaClient` for local Ollama fallback.
- Refactored `core/classifier.py` to support provider routing and failover.
- Wired `app.py` to read classifier provider settings from environment variables.
- Updated `.env.example` with DashScope and Ollama configuration entries.
- Cleaned and normalized `requirements.txt`, including `tenacity` and `httpx`.
- Added deterministic unit tests for primary success, failover, malformed JSON, and safe fallback.

## Key Decisions
- Primary classifier provider: DashScope.
- Fallback classifier provider: Ollama.
- Default classifier model: `qwen-14b-chat`.
- Fallback behavior on total failure: return `General` with confidence `0.0`.

- `python -m pytest tests/test_api.py -q` → 3 passed

## Notes
- Gemini support remains available as a rollback path via `CLASSIFIER_PROVIDER=gemini`.
- The classifier now tolerates malformed JSON or provider exceptions by failing over before returning the safe fallback.
