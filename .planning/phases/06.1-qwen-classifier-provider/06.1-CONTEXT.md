# Phase 6.1 Context - Qwen Classifier Provider

## Why This Phase Exists

Introduce an additional LLM provider for query classification so the system is not coupled to Gemini only.

## Locked Decisions

1. Primary provider: DashScope.
2. Backup provider: Ollama.
3. Model: qwen-14b-chat.
4. Numbering: Phase 6.1 (kept isolated from shipped v1.2 scope).

## Scope

- Classifier path only (user query classification).
- Provider abstraction for Gemini and Qwen routing.
- Config switches and fallback behavior for classifier runtime.

## Out of Scope

- Full generation model migration.
- Frontend work and chat history features.
