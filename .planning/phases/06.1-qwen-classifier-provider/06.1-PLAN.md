# Plan 06.1: Qwen Classifier Provider (DashScope Primary, Ollama Backup)

## Objective
Add `qwen-14b-chat` support for query classification with provider failover: DashScope first, Ollama second, and safe fallback to `General` when all providers fail.

## Must-Haves
- Classifier defaults to DashScope with `qwen-14b-chat`.
- On provider errors/timeouts, classifier retries/fails over to Ollama.
- API behavior and response schema remain unchanged.
- Provider selection and fallback toggles are environment-configurable.

## Tasks

### 1. Add provider contract and Qwen clients
<action>
Create `tools/llm_client_base.py` with a small provider interface used by classifier clients:
- `class BaseLLMClient` with async method `generate_content_async(prompt: str, **kwargs)`.

Create `tools/qwen_dashscope_client.py`:
- Implement `QwenDashScopeClient(BaseLLMClient)` using async HTTP calls (HTTP endpoint for DashScope-compatible chat completion).
- Read config from env: `DASHSCOPE_API_KEY`, optional `DASHSCOPE_BASE_URL`, `QWEN_MODEL` (default `qwen-14b-chat`).
- Normalize response into object with `.text` so classifier parser logic stays compatible.
- Add timeout and retry behavior for transient errors.

Create `tools/qwen_ollama_client.py`:
- Implement `QwenOllamaClient(BaseLLMClient)` using Ollama chat/generate HTTP API.
- Read config from env: `OLLAMA_BASE_URL`, `OLLAMA_MODEL` (default `qwen-14b-chat`).
- Normalize response into object with `.text`.
</action>

- Extend `LegalQueryClassifier.__init__` to accept:
  - `provider: str = "dashscope"`
  - `fallback_provider: str = "ollama"`
  - `model_name: str = "qwen-14b-chat"`
  - `api_key: Optional[str] = None`
- Add internal provider factory that can build:
  - `GeminiClient` when provider is `gemini`
  - `QwenDashScopeClient` when provider is `dashscope`

- In `classify`, attempt primary provider first; on provider/network/parsing failure, attempt backup provider.
- If both fail, return safe fallback: `domains=["General"]`, `confidence=0.0`, `is_explicit_filter=False`.
- Keep JSON cleanup/parsing compatible with existing behavior.

## Risks and Mitigation
- DashScope/Ollama schema drift: keep response normalizer in each client.
- Failover latency: use strict per-provider timeout before switching.
- Environment drift: keep Gemini path intact for rollback by setting `CLASSIFIER_PROVIDER=gemini`.
